üìö Recap: Multi-Programming Problem

Problem from Last Lesson:

In Multi-Programming, when CPU is assigned a task, it processes it until the task is completely finished and exits.

During I/O operations, CPU remains in IDLE state instead of processing other tasks.

The Solution: Scheduling Concept

Scheduling = Planning how to complete work with:

  • Minimum time
  • Minimum effort
  • Maximum efficiency
If 10 students are given the same task, all will complete it but in different times. The one who finishes fastest has better "processing speed" - that's scheduling optimization!

üîÑ How Multi-Tasking Works

Imagine 4 guests come for dinner, each eating 4 rotis. You have only 8 rotis ready. Instead of serving 2 people fully and making others wait, you give 1-1 roti to everyone, then repeat. Everyone feels served!

The Switching Mechanism:

CPU doesn't fully complete one program before moving to next.

Instead, it switches between programs very quickly:

  • Process P1 for a bit ‚Üí Switch to P2
  • Process P2 for a bit ‚Üí Switch to P3
  • Process P3 for a bit ‚Üí Switch to P4
  • Back to P1 ‚Üí Continue the cycle

Key Insight:

This switching is SO FAST that it appears all programs are running simultaneously!

Reality: Programs run alternatively (one by one)

Perception: Programs appear to run simultaneously

‚ö° Multi-Processing Operating System

The Solution: Hardware Advancement!

Multi-Processing OS uses MORE THAN ONE CPU/Processor in a single computer system.

Think of your family: Dad is the main "processor" earning for everyone. But what if he gets sick? That's why YOU want to become independent - to become another "processor" and share the load!

Key Features:

  • Uses 2 or more CPUs in single computer
  • Multiple processors share: Computer Bus, Clock, Memory
  • Resources are shared between processors
  • If one processor fails, others continue working
  • Hardware level advancement (not just software)
Multi-Processing = Multiple Processors (Hardware upgrade)
Multi-Programming/Multi-Tasking = Single Processor (Software upgrade)

‚è±Ô∏è Real-Time Operating System (RTOS)

Main Characteristic:

Tasks are given a FIXED TIME LIMIT to complete.

Expected to finish within that time limit!

Like an exam: You have 2 hours to solve 150 questions. The examiner doesn't need students who can solve 150 questions - they need students who can solve them IN 2 HOURS!

Key Definition:

In RTOS, as soon as input is entered, system provides IMMEDIATE or QUICK response.

RTOS = They are FAST!

Examples of RTOS:

  • üèß ATM Machines - Millions transacting simultaneously, yet instant response
  • üöÇ Railway Reservation - Lakhs booking at same time, immediate confirmation
  • üö¶ Traffic Lights - Must switch exactly on time
  • üöó Car Airbags - Must deploy instantly on collision

Soft Real-Time OS

  • ‚úÖ You can RE-ATTEMPT
  • ‚úÖ No major disaster
  • ‚úÖ System continues working

Examples:

  • ATM transactions (retry possible)
  • Railway reservations
  • Online examinations

Hard Real-Time OS

Time limit is fixed, AND if task fails:

  • ‚ùå NO re-attempt possible
  • ‚ùå Major disaster/damage occurs
  • ‚ùå Critical failure

Examples:

  • Traffic lights (wrong timing = accidents)
  • Car airbags (late = injury/death)

Soft RTOS: Failure is recoverable (can retry)
Hard RTOS: Failure is catastrophic (no second chance)

üìä CPU Scheduling - Introduction

What is CPU Scheduling?

The strategy used by CPU to process inputs and provide outputs as quickly as possible with minimum effort.

Important Terms to Know:

1. CPU Utilization

Keep the CPU as busy as possible. CPU is the brain - it should always be working!

2. Throughput

Use CPU to maximum capacity. Get maximum outputs from it!

‚è∞ CPU Scheduling - Time Terms

Imagine visiting a doctor's clinic. You arrive at 10 AM, doctor calls you at 12:30 PM, treats you till 1 PM, and you leave at 2 PM. Let's understand all times using this example!

Arrival Time (AT)

Time when process enters the system (RAM)

Example: You arrived at clinic at 10:00 AM

Burst Time (BT)

Time taken by CPU to process the task

Example: Doctor treated you for 30 minutes (12:30 to 1:00 PM)

Completion Time (CT)

Time when process finishes execution

Example: Your treatment completed at 1:00 PM

Exit Time

Time when process leaves the system

Example: You left clinic at 2:00 PM

11 / 16

‚è∞ Important Formulas

Turn Around Time (TAT)

TAT = Completion Time - Arrival Time

Total time from arrival to completion

Waiting Time (WT)

WT = Turn Around Time - Burst Time

Time spent waiting (not being processed)

Response Time (RT)

RT = CPU First Arrival - Original Arrival Time

Time until CPU first responds to the process

Doctor's Clinic Example:

  • Arrival Time: 10:00 AM
  • Response Time: 2.5 hours (10 AM to 12:30 PM)
  • Burst Time: 30 minutes (12:30 to 1:00 PM)
  • Completion Time: 1:00 PM
  • TAT: 3 hours (10 AM to 1 PM)
  • Waiting Time: 2.5 hours (TAT - BT = 3 - 0.5)

üîÄ Scheduling Algorithm Types

Scheduling algorithms are of two types:

Preemptive

Task is processed bit by bit

  • Start ‚Üí Process a little ‚Üí Switch
  • Come back ‚Üí Process more ‚Üí Switch
  • Continue until complete

Prevents CPU Monopolization

Non-Preemptive

Task runs start to finish

  • Start ‚Üí Process completely ‚Üí Exit
  • Won't leave CPU until done
  • Other tasks must wait

CPU Monopolization occurs

Q: Which algorithm prevents CPU monopolization?
A: Preemptive Algorithm

Key Point: In Non-Preemptive algorithms, Waiting Time = Response Time

üìã First Come First Serve (FCFS)

FCFS Algorithm

Type: Non-Preemptive

Rule: Whoever arrives first gets served first!

Understanding with Ready Queue & Gantt Chart:

Ready Queue (RAM)

Where processes wait before going to CPU

Shows the order of processes waiting

Gantt Chart (CPU)

Shows how CPU processes each task

Visual representation of execution order

Remember:

  • Input goes to RAM first, then to CPU
  • First process to arrive has Waiting Time = 0
  • In Gantt Chart: Left side = Start time, Right side = Completion time
