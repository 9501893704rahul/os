‚öôÔ∏è Functions of Operating System - Part 1

1. Processor Management

The processor (CPU) is the most expensive component in a computer.

OS ensures maximum utilization of the processor.

2. Resource Manager

When multiple programs run simultaneously, OS manages:

  • When to give resources to which program
  • When to take back resources
  • All programs ultimately run on CPU and RAM

3. Memory Management

OS manages all types of memory:

  • RAM - Primary memory
  • Cache - Fast temporary memory
  • Hard Disk Drive - Secondary storage

What is a "Batch"?

Batch = Group

For a group to exist, two conditions must be met:

  • More than one item (multiple items)
  • Same type (all items must be similar)

üì¶ Batch Processing - How It Works

Batch Processing Workflow

Input 1
Input 2
Input 3

‚Üì Wait until batch is formed ‚Üì

CPU Processes Entire Batch

‚ùå Problems with Batch Processing:

  • Single input must WAIT until batch is formed
  • CPU remains IDLE while waiting for batch
  • Costly processor not utilized properly
  • Only processes same type of jobs

Real-Life Example: NEFT

NEFT
National Electronic Fund Transfer

NEFT uses batch processing - transactions are collected and processed in batches (approximately 48 transactions per batch). That's why NEFT takes 30-60 minutes, unlike IMPS/RTGS which are instant.

Bank account opening is also an example of Batch Processing - forms are collected and processed together!

üîÑ Multi-Programming - The Remaining Problem

Two Types of Programs:

Type 1: Start ‚Üí End ‚Üí Exit

Program starts, completes fully, then exits.

Example: Playing a movie - starts and plays till end

Type 2: Start ‚Üí Pause ‚Üí Continue ‚Üí End

Program needs I/O operations in between.

Example: Filling a form - open form, wait for input, submit

The Problem Scenario:

P1 (Complete)
‚Üí
CPU
‚Üí
Exit ‚úÖ

Then P2 comes (needs I/O operation)...

P2 (Partial)
‚Üí
CPU
‚Üí
Back to RAM (waiting)

CPU becomes IDLE while P2 waits!

‚ùå Problem:

When a program goes back to RAM for I/O operation, CPU waits for it instead of processing other programs (P3, P4...).

Result: P3, P4 have very long waiting time!

RAM = Waiting Room

Programs wait in RAM before going to CPU and return to RAM during I/O operations.
