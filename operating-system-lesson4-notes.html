üìã Shortest Job First (SJF) Scheduling

What is SJF?

Mode: Non-Preemptive

Rule: Process with shortest burst time gets served first!

Imagine a hotel owner says: "Whoever eats less will be served first, whoever eats more will be served later." That's SJF - shortest job (smallest burst time) gets priority!

Key Points:

  • Smaller burst time = Higher priority
  • Larger burst time = Lower priority
  • Need comparison to determine which job is shorter
  • First job has no comparison - it runs immediately

When Burst Times are Equal:

If two jobs have same burst time, priority is given based on Arrival Time (who came first).

üîç SJF Key Observations

Important Finding:

In SJF (Non-Preemptive), Waiting Time = Response Time for all processes!

This is true for ALL Non-Preemptive scheduling algorithms (FCFS, SJF, etc.)

Why WT = RT in Non-Preemptive?

Think of visiting a doctor for a simple fever - you go, get treated, and leave. But if you're hospitalized for 15 days, the doctor visits you multiple times. In Non-Preemptive, it's like the simple visit - one continuous session!

Conclusion:

Non-Preemptive Scheduling: WT = RT (always)

Preemptive Scheduling: WT ‚â† RT (usually different)

üîÑ Round Robin Scheduling

What is Round Robin?

Mode: Preemptive

Key Concept: Each process gets a small unit of CPU time called Time Quantum

How It Works:

  • Each process gets a fixed time slice (Time Quantum)
  • After time quantum expires, process is preempted
  • Process goes back to end of Ready Queue
  • Next process in queue gets CPU
  • Cycle continues until all processes complete
Like a hotel that says: "No matter how hungry you are, we'll give only 2 rotis at a time. Want more? Go back in line!" Everyone gets fair share!

Time Quantum: Usually 10-100 milliseconds

This prevents any single process from monopolizing the CPU!

‚öñÔ∏è Preemptive vs Non-Preemptive Summary

Non-Preemptive (FCFS, SJF)

  • Process runs start to finish
  • No switching mid-process
  • CPU monopolization possible
  • WT = RT (always)

Preemptive (Round Robin)

  • Process runs in small chunks
  • Frequent switching
  • Fair CPU distribution
  • WT ‚â† RT (usually)
Non-Preemptive: Like visiting doctor for fever - one visit, done.
Preemptive: Like being hospitalized - doctor visits multiple times over days.
Q: Which scheduling prevents CPU monopolization?
A: Preemptive Scheduling (like Round Robin)

üñ®Ô∏è Spooling Concept

When is Spooling Used?

When User's requirement > Machine's execution power

SPOOL Full Form:

Simultaneous Peripheral Operations Online

Think of a barber shop on Sunday - many customers, but barber can only cut one person's hair at a time. A queue forms! Similarly, when input is more than execution capacity, a queue (spool) forms.

Real-Life Examples:

  • üé´ Railway ticket counter - many people, slow processing
  • üíá Barber shop on weekends - high demand, limited capacity
  • üñ®Ô∏è Printer - multiple print jobs, one printer

Key Insight: If input speed = execution speed, no queue forms!

üñ®Ô∏è Spooling - Printer Example

Scenario: Computer Lab with Shared Printer

5 Computers (A, B, C, D, E) connected to 1 Printer via Network

1 Computer A sends 20-page print job
2 While printing, Computer B sends 25-page job
3 Computer C sends 15-page job
4 Computer D sends 30-page job

What Happens?

  • Jobs form a queue (First Come First Serve)
  • A's 20 pages won't stop for B's larger job
  • C's smaller job won't jump the queue
  • This queue area is called SPOOL
Printer is the best example of SPOOLING and is a shareable device in network!

üñ®Ô∏è Spooling - Important Points

To Cancel a Print Job:

Go to Spooler (Print Queue Manager)

Key Exam Points:

Q: Which device is shareable in a network?

A: Printer

Q: What is SPOOL?

A: Simultaneous Peripheral Operations Online - A memory area where jobs wait in queue

Q: Spooling uses which scheduling?

A: First Come First Serve (FCFS)

Printer Characteristics:

  • Shares resources with multiple computers
  • Uses FCFS for job processing
  • Creates a spool (queue) for pending jobs
  • Won't work without paper or ink!
